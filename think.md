You're not building a mentoring app. You're building a UI/UX tool that uses AI — something that makes UI/UX work faster, smarter, or possible in new ways. Here are genuinely unique ideas:

Option 1 — "AI UX Auditor" (Most Unique + High Company Value)
What: Upload any screen (screenshot or your Angular app URL) → AI audits it and returns:

Accessibility violations with severity
UX principle violations (contrast, cognitive load, hierarchy)
Auto-generated Angular fix code for each issue
Before/after visual comparison
Why unique: Every tool flags issues. None generate the actual fix code scoped to your design system.

Company benefit: Automated QA for every UI release.

Option 2 — "Talk to Your Design System"
What: A chat interface over your company's design system:

"Show me how to build a data table with filters" → AI returns the exact component code using your real tokens/components
"What color should I use for destructive actions?" → AI answers from your brand rules
Designers and devs both use it — no more Confluence hunting
Why unique: Design systems are powerful but nobody reads them. AI makes them conversational.

Company benefit: Faster onboarding, fewer design inconsistencies.

Option 3 — "Natural Language → Wireframe → Code" (Ties to your existing plan)
What: 3-step flow:

User describes a screen in plain English
AI generates a visual wireframe (rendered in browser, not an image)
One click converts wireframe to Angular component code
Why unique: Most tools stop at step 2. The Angular code output scoped to your components is the differentiator.

Option 4 — "AI Persona + Journey Generator"
What: Describe your product → AI generates:

User personas with pain points
User journey maps (visual, interactive)
Suggested UI patterns per persona
Exports to a shareable link
Why unique: UX research takes weeks. This gives a starting point in minutes.

Build Option 2 (Talk to Your Design System) for Phase 1, then add Option 1 (UX Auditor) as Phase 2.
